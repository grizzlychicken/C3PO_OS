****Code Structure*****
Animation Control
Switch is set in events loop for ANIMATION_MODE.  Read switch is processed in Motion Control loop and calls AnimationControl function in GeneratedCodeAnimations namespace.  Here setup is called to register servos and bottango loop is called.  Movement processing then occurs in BottangoArduinoCallbacks function and movement processed in onearlyloop function.
Have detaches servos on entry to loop of RC servos and bottango animation servos to get a clean transfer.  No detachement of bottango servos as yet on exit from ANIMATION_MODE.

RC Control
Event sloop processes switch for RC-MODE.  Switches and buttons processed in events loop.  Elbow and wrist servos are on off switches using trim switches. Functions for this are in the motion control section and switches are read and triggered in the Events section.
**need to smooth elbow and wrist movements and blend with other movements in RC mode****
Servos are attached manually in RCMODE with momentary switch.

First 3 way switch switches from:
-off
-RC drive mode - head functions still actuate
-RC control mode- all servos are active

Second 3 way switch switches from:
- RC animations Off
- RC animations Auto mode- does auto head functions
- Rc Animations Control mode

Open main Controller program in VS code.  So Teensy_Main and Teensy_Slave are the 2 project files.
Open GeneratedCodeAnimations.ccp files in vs code one at a time.  Copy everything below  // setup and replace the existing code in the generatedcodeanimations.ccp file.
